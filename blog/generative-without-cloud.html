Generative ambient without the cloud

Endel sells generative as an AI feature with a subscription. We argue it's a synthesis problem solved by ~250 lines of Swift — on-device, free.

The promise of "generative ambient" is that no two minutes ever sound exactly the same. The actual mechanism most apps use is narrower than the marketing suggests: cross-fade a handful of sample loops at slightly different points, modulate the volume of each by a few independent envelopes, and let the listener's pattern-matching brain do the rest.

We thought we could ship the same effect with ~5kB of code and zero samples. Here's how.

1. A modulator is a function of time

We define a handful of modulator types — sine, triangle, random walk, beating — each returning a value in [-1, 1] for a given elapsed time. 

case sinusoid(periodSec: Double)   // sin(2π · t / period)
case triangle(periodSec: Double)
case randomWalk(stepSec: Double)
case beating(periodSecA: Double, periodSecB: Double)
case hold                              // always returns 0

2. A recipe attaches a modulator to a layer

Each ambient layer gets one recipe: which modulator drives it, how much depth (the half-range of gain swing), an optional pan depth, and a phase offset so neighboring layers don't peak at the same moment. 

struct LayerRecipe {
  var modulator: Modulator
  var depth: Float       // ±depth gain swing around the baseline
  var panDepth: Float    // optional ±panDepth pan swing
  var phaseOffset: Double
}

3. A pattern is a list of recipes

Picking a pattern means picking how the layers evolve. The focus-flow pattern uses three slow sinusoids at incommensurate periods (90s, 120s, 150s) with offset phases. The dream-shift pattern uses three random walks for surreal layer juxtapositions. The ambience-shimmer pattern uses three beating pairs at fast/medium/slow scales. 

focus-flow:
  layer[0]: sinusoid(90s),  depth=0.10, phase=0
  layer[1]: sinusoid(120s), depth=0.12, phase=0.33
  layer[2]: sinusoid(150s), depth=0.08, phase=0.66
  baseNoise: sinusoid(180s), depth=0.04

4. The composer is a 1Hz timer

Every second (or twice a second, depending on the pattern), the composer evaluates each recipe and applies the result to the live audio graph. Snapshot the layer baselines at start time so the user's chosen mix is the center of evolution — we never overwrite their dial, we drift around it. 

tick() {
  let t = elapsedSec
  for (layer, recipe) in zip(layers, pattern.recipes) {
    let v = recipe.modulator.value(at: t + recipe.phaseOffset)
    let gain = clamp01(baseline[layer] + v * recipe.depth * intensity)
    engine.setLayerGain(layer.id, gain)
  }
}

Why this matches "endless and never repeats"

Three sinusoids at incommensurate periods (90, 120, 150) produce a global pattern that only fully repeats every LCM(90, 120, 150) / 60 ≈ 30 minutes. Add random walks and the period is infinite. Add pan modulation and you get the spatial drift that gives Endel's marketing copy its "spatial AI" flavor. None of it requires a model checkpoint.

CPU cost

On an iPhone 17 Pro Max, the composer ticking at 1Hz across six ambient layers + the base noise consumes under 0.4 % of one CPU core. The audio synthesis itself (pink noise via Paul Kellet's filter, brown via a leaky integrator) consumes another ~2 %. A cloud generative engine, by contrast, requires network round-trips, model load, and audio streaming — typically 10-50× more battery for the same listener perception.

Where the trick lives

The trick isn't in the math — it's in the tuning. Picking depths small enough that the listener doesn't notice movement, but large enough that they don't get bored, is taste, not algorithm. We spent more time tuning the eight shipped patterns than we spent writing the engine. The engine is open in spirit; the curation is the product.
